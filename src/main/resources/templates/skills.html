Hibernate

Подробнее...

Hibernate

Реализация JPA спецификации, предназначенная для решения задач объектно-реляционного отображения.

Из всех известных мне фреймворков, думаю, Hibernate я знаю лучше всего. Liquibase

Liquibase

Apache Wicket

Подробнее...

Apache Wicket

Веб-фреймворк использующий подход, основанный на использовании компонентов. Пожалуй, это фреймворк, в применении которого я имею больше всего опыта.

В моей работе большинство проектов были построены с использованием этого фреймворка. Наиболее сложными задачами, с которыми я сталкивался, была интеграция Wicket'а с компонентами, написанными на JS (№ календари, карты).

Также я учавствовал в доработке моделей для работы с сущностями Hibernate в рамках библиотеки wicket-autodao.

По моему опыту Wicket хорошо подходит для простых сайтов. Для более глобальных проектов использование этого фреймворка ведет к быстрой деградации кодовой базы и появлению большого количества спагетти-кода. Этот фреймворк требует твердых методик кодирования.

ELK

Подробнее...

ELK

ELK стек (Elasticsearch + Logstash + Kibana) позволяет организовать эффективный сбор и мониторинг логов. Логи собираются с помощью beats (наиболее часто FileBeat), обрабатываются в Logstash и сохраняются в БД Elasticsearch. Впоследствии с логами можно работать через графический интерфейс Kibana.

В рамках домашнего проекта я организовывал сбор логов с Kubernetes кластера. Разворачивал кластер Elasticsearch из 3 нод, настраивал ILM-политики, описывал пайплайн в Logstash, работал с Kibana.

Solidity

Мной был разработан смарт-контракт обменника эфира на токены (и наоборот) при привязке цены к внешнему фактору (курс эфира к рублю). Для получения курса доллара изначально был использован оракул Oraclize, однако, затраты на содержание обменника оказались для заказчика неприемлемыми. В итоге было принято решение реализовать свой сервис-оракул. Это позволило сократить затраты на содержание обменника в 4 раза.
